127 /**
128  * \defgroup hal_GeneralSerial Serial Configuration Functions
129  *
130  * # Defined behavior
131  * * ::serial_init initializes the ::serial_t
132  * * ::serial_init sets the default parameters for serial peripheral (9600 bps, 8N1 format)
133  * * ::serial_init configures the specified pins
134  * * ::serial_free releases the serial peripheral
135  * * ::serial_baud configures the baud rate
136  * * at least 9600 bps the baud rate must be supported
137  * * ::serial_format configures the transmission format (number of bits, parity and the number of stop bits)
138  * * at least 8N1 format must be supported
139  * * ::serial_irq_handler registers the interrupt handler which will be invoked when the interrupt fires.
140  * * ::serial_irq_set enables or disables the serial RX or TX IRQ.
141  * * If `RxIrq` is enabled by ::serial_irq_set, ::serial_irq_handler will be invoked whenever
142  * Receive Data Register Full IRQ is generated.
143  * * If `TxIrq` is enabled by ::serial_irq_set, ::serial_irq_handler will be invoked whenever
144  * Transmit Data Register Empty IRQ is generated.
145  * * If the interrupt condition holds true, when the interrupt is enabled with ::serial_irq_set,
146  * the ::serial_irq_handler is called instantly.
147  * * ::serial_getc returns the character from serial buffer.
148  * * ::serial_getc is a blocking call (waits for the character).
149  * * ::serial_putc sends a character.
150  * * ::serial_putc is a blocking call (waits for a peripheral to be available).
151  * * ::serial_readable returns non-zero value if a character can be read, 0 otherwise.
152  * * ::serial_writable returns non-zero value if a character can be written, 0 otherwise.
153  * * ::serial_clear clears the ::serial_t RX/TX buffers
154  * * ::serial_break_set sets the break signal.
155  * * ::serial_break_clear clears the break signal.
156  * * ::serial_pinout_tx configures the TX pin as an output (to be used in half-duplex mode).
157  * * ::serial_set_flow_control configures serial flow control.
158  * * ::serial_set_flow_control sets flow control in the hardware if a serial peripheral supports it,
159  * otherwise software emulation is used.
160  * * ::serial_tx_asynch starts the serial asynchronous transfer.
161  * * ::serial_tx_asynch writes `tx_length` bytes from the `tx` to the bus.
162  * * ::serial_tx_asynch must support 8 data bits
163  * * The callback given to ::serial_tx_asynch is invoked when the transfer completes.
164  * * ::serial_tx_asynch specifies the logical OR of events to be registered.
165  * * ::serial_rx_asynch starts the serial asynchronous transfer.
166  * * ::serial_rx_asynch reads `rx_length` bytes to the `rx` buffer.
167  * * ::serial_rx_asynch must support 8 data bits
168  * * The callback given to ::serial_rx_asynch is invoked when the transfer completes.
169  * * ::serial_rx_asynch specifies the logical OR of events to be registered.
170  * * ::serial_rx_asynch specifies a character in range 0-254 to be matched, 255 is a reserved value.
171  * * If SERIAL_EVENT_RX_CHARACTER_MATCH event is not registered, the `char_match` is ignored.
172  * * The SERIAL_EVENT_RX_CHARACTER_MATCH event is set in the callback when SERIAL_EVENT_RX_CHARACTER_MATCH event is
173  * registered AND `char_match` is present in the received data.
174  * * ::serial_tx_active returns non-zero if the TX transaction is ongoing, 0 otherwise.
175  * * ::serial_rx_active returns non-zero if the RX transaction is ongoing, 0 otherwise.
176  * * ::serial_irq_handler_asynch returns event flags if a transfer termination condition was met, otherwise returns 0.
177  * * ::serial_irq_handler_asynch takes no longer than one packet transfer time (packet_bits / baudrate) to execute.
178  * * ::serial_tx_abort_asynch aborts the ongoing TX transaction.
179  * * ::serial_tx_abort_asynch disables the enabled interupt for TX.
180  * * ::serial_tx_abort_asynch flushes the TX hardware buffer if TX FIFO is used.
181  * * ::serial_rx_abort_asynch aborts the ongoing RX transaction.
182  * * ::serial_rx_abort_asynch disables the enabled interupt for RX.
183  * * ::serial_rx_abort_asynch flushes the TX hardware buffer if RX FIFO is used.
184  * * Correct operation guaranteed when interrupt latency is shorter than one packet transfer time (packet_bits / baudrate)
185  * if the flow control is not used.
186  * * Correct operation guaranteed regardless of interrupt latency if the flow control is used.
187  *
188  * # Undefined behavior
189  * * Calling ::serial_init multiple times on the same `serial_t` without ::serial_free.
190  * * Passing invalid pin to ::serial_init, ::serial_pinout_tx.
191  * * Calling any function other than ::serial_init on am uninitialized or freed `serial_t`.
192  * * Passing an invalid pointer as `obj` to any function.
193  * * Passing an invalid pointer as `handler` to ::serial_irq_handler, ::serial_tx_asynch, ::serial_rx_asynch.
194  * * Calling ::serial_tx_abort while no async TX transfer is being processed.
195  * * Calling ::serial_rx_abort while no async RX transfer is being processed.
196  * * Devices behavior is undefined when the interrupt latency is longer than one packet transfer time
197  * (packet_bits / baudrate) if the flow control is not used.
198  * @{
199  */
